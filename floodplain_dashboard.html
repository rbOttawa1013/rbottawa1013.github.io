Canadian flood plain risk dashboard

Enter a Canadian postal code to view a map-centred flood-risk profile with public base maps, federal reference layers, and province-aware flood data links where public APIs are available.

Alpha prototype · Not an official Government of Canada production service · Date modified: 2026-06-25

Before you start

This tool is for exploration and education. It does not determine whether a property is inside a regulated flood plain. Flood mapping is authoritative only when confirmed by the responsible municipality, conservation authority, province, territory or federal program.

Interpretation rule

Visible water, low elevation or nearby flood layers do not prove legal flood-plain status. Use this dashboard to identify which authoritative source to check next.

Map and public layers

The map starts at the resolved postal-code centroid. Use the buttons to change the base map, then choose overlays for federal reference layers and provincial or municipal hazard services.

Uses NRCan/OpenMaps-style WMS endpoints where available, with visible fallback if unavailable.

Sources, APIs and limitations

The page uses public services and defensive fallbacks. If a public map service blocks cross-origin browser access, is unavailable, or has a changed layer name, the dashboard reports that condition rather than silently presenting mock data as authoritative.

Accessibility, privacy and data-quality gates

  • Privacy: the form requests only a postal code and does not store the value.
  • Accessibility: the page includes skip navigation, visible focus states, semantic landmarks, labelled controls, an error summary and print support.
  • Data quality: the page distinguishes live public API data, reference layers, unavailable layers and non-authoritative guidance.
  • Security: no API keys, tokens or credentials are embedded.
  • Limitations: production deployment requires departmental validation of Google tile terms, official map service endpoints, accessibility testing, privacy review and legal disclaimers.
