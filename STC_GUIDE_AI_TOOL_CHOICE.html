AI Hub Responsible AI tools How to choose a Statistics Canada AI tool

Prototype status: This page converts the canonical StatCan AI tools decision table into an accessible decision-support page. Confirm tool approvals, data-class permissions, licensing, and production-readiness with the relevant StatCan governance authority before operational use.

1. Purpose Use this page to select an appropriate AI tool by scenario, data sensitivity, risk level, and intended output. It is designed for internal AI Hub or repository use as a routing aid, not as a substitute for security, privacy, legal, translation, statistical, or release approval.

Bottom line

  • General work: use M365 Copilot when available; use Copilot Web Chat only for low-risk, non-sensitive work.
  • Translation: use GCTranslate for informal internal translation only; do not use it for official or publication-ready outputs.
  • Rapid non-sensitive chatbot: use Copilot Studio for low-risk, non-Protected B use.
  • Sensitive or Protected B: use StatCan AI Assistant and, where scalable backend processing is needed, AIEP.
  • Build or scale AI: use AIEP, with a Coding Assistant for development acceleration where approved.

    5. Decision logic

    General productivity

    Use: M365 Copilot when available.

    Fallback: Copilot Web Chat for low-risk, non-sensitive work only.

    Translation

    Use: GCTranslate for informal internal translation.

    Do not use: GCTranslate for official, public, publication-ready, or sensitive outputs.

    Sensitive / Protected B

    Use: StatCan AI Assistant as the primary interface.

    Use AIEP: backend, scalable, governed, or technical processing.

    Low-risk chatbot

    Use: Copilot Studio for quick, low-code, non-Protected B agents.

    Do not use: Copilot Studio for Protected B or production-sensitive workflows.

    Protected B chatbot

    Use: StatCan AI Assistant as the front door.

    Use: AIEP as backend. Do not use Copilot Studio.

    Build / scale AI

    Use: AIEP for hosting, orchestration, APIs, pipelines, storage, and model operations.

    Use: Coding Assistant for development acceleration where approved.

    8. Detailed tool guidance
    Low-risk and non-sensitive work

    Use Copilot Web Chat, M365 Copilot, StatCan AI Assistant, or low-risk prototyping tools only when the content does not include sensitive, protected, confidential, personal, or controlled information.

    • Use generic context and remove identifiers.
    • Do not paste internal records unless the tool is approved for that data class.
    • Validate all factual claims before reuse.
    M365 productivity work

    Use M365 Copilot for work embedded in Word, Excel, PowerPoint, Outlook, Teams, and other Microsoft 365 workflows where licensing and governance permit.

    • Confirm whether the file or mailbox content is appropriate for the tool configuration.
    • Do not assume all M365 content is automatically suitable for AI processing.
    • Review generated summaries, formulas, slides, or messages before sharing.
    Translation and official languages

    Use GCTranslate only for informal internal translation where the content is not sensitive and not intended as an official or publication-ready output.

    • For official translation, use the approved official languages and translation process.
    • Check terminology, meaning equivalence, tone, and legal or statistical wording.
    • Do not use AI translation for sensitive, protected, or public-release content unless specifically approved.
    Protected B and sensitive statistical work

    Use StatCan AI Assistant as the primary governed interface where the data class and use case are approved. Use AIEP for backend, scalable, model, or pipeline processing.

    • Do not use Copilot Web Chat, GCTranslate, or Copilot Studio for Protected B data.
    • Confirm the data class, retention, access control, and logging requirements.
    • Apply statistical confidentiality, disclosure control, and quality validation where relevant.
    Chatbots, agents, and reusable workflows

    Use Copilot Studio for rapid low-code agents only when the context is low-risk and non-Protected B. Use StatCan AI Assistant and AIEP for governed, sensitive, Protected B, or production contexts.

    • Define intended users, data sources, guardrails, escalation, and ownership.
    • Test for hallucination, privacy leakage, prompt injection, accessibility, and bilingual behaviour.
    • Assign monitoring, support, and change-management responsibilities before launch.
    Custom AI, pipelines, model hosting, and coding

    Use AIEP for full-stack AI solution development, hosting, orchestration, data ingestion, APIs, storage, and model operations. Use a Coding Assistant for developer productivity where approved.

    • Never paste credentials, secrets, tokens, keys, or unapproved data into coding tools.
    • Review generated code for security, accessibility, maintainability, licensing, and correctness.
    • Use proper source control, testing, architecture review, and operational readiness checks.
    Production and official outputs

    Production-grade AI deployment requires governed workflows. AIEP is the primary platform for production-grade AI infrastructure, while StatCan AI Assistant may serve as an approved interface layer.

    • Do not treat AI-generated output as final without validation and approval.
    • Apply accessibility, official languages, privacy, security, communications, statistical quality, and records requirements.
    • Document the tool, model, data sources, prompts, assumptions, validation results, and approval path.
