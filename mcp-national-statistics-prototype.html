MCP for authoritative national statistics

A working prototype showing how an AI assistant can use the Model Context Protocol to discover, validate and cite authoritative national statistics sources instead of inventing or scraping statistical claims.

Static HTML/CSS/JavaScript prototype · No login · No backend · No external API calls · Demo data only

What this demonstrates

MCP is shown as a controlled interface between an AI host and a national statistical organization. The host discovers approved tools, reads approved metadata resources, validates the authority of a source, calls tools with structured arguments, and receives structured results with source, quality and limitation notes.

How MCP maps to official statistics

A national statistical organization can expose official statistical capabilities as controlled MCP resources, tools and prompt templates. This makes the source of truth explicit and gives AI systems a safer path to statistical metadata, release status, quality notes and citation text.

MCP architecture for authoritative national statistics

The AI host communicates through an MCP client to an NSO MCP server that validates requests against an authoritative source registry.

Tools

Executable actions such as validating a source, listing dataflows, comparing geographies, checking a claim and creating citation-ready summaries.

Resources

Read-only context such as dataset metadata, official-release rules, source authority status, quality frameworks and disclosure warnings.

Prompts

Reusable templates for statistical briefing notes, source verification, metadata checks, plain-language summaries and misuse warnings.

      Authoritative source registry

      The MCP server should not treat every dataset link as authoritative. It should validate publisher, official status, release state, geography, reference period, method and citation requirements before returning a result.

      MCP server contract

      The contract below shows what a national statistics MCP server could expose. A production implementation would enforce authentication, authorization, rate limits, audit logs and approved data domains.

      Privacy, security and statistical quality gates

      Before the tool call

      • Authenticate the user and apply least-privilege authorization.
      • Expose only approved aggregate public or permissioned datasets.
      • Require official-source ID, geography, reference period, unit, release state and citation text.
      • Block personal, confidential, protected or unpublished data in public assistant contexts.

      After the tool call

      • Return publisher, source URL, methodology, quality notes, limitations and update status.
      • Do not infer causes from descriptive statistics without supporting data.
      • Log tool calls for audit without storing sensitive user prompts unnecessarily.
      • Mark AI summaries as draft text requiring human and statistical-methodology verification.

      Responsible-use boundary

      This prototype allows an AI assistant to support statistical discovery and explanation. It does not automate eligibility, enforcement, benefit, compliance or any decision about people. Production use requires departmental privacy, security, accessibility, official languages and statistical-methodology review.
