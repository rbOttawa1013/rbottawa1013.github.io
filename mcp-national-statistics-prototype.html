MCP national statistics protocol simulator

Use this working browser prototype to see how an AI assistant could discover, read and call national statistics tools through the Model Context Protocol.

Static HTML/CSS/JavaScript prototype · No login · No backend · No external API calls · Demo data only

What this demonstrates

MCP is shown as a safe interface between an AI host and a national statistics server. The AI host does not scrape pages or guess data. It discovers approved tools, reads approved metadata resources, calls a tool with structured arguments, and receives a structured response with source and quality notes.

How MCP maps to national statistics

A national statistical organization can use an MCP server to expose only approved statistical capabilities. The server can make dataset metadata available as resources, analysis actions available as tools, and reusable plain-language workflows available as prompts.

MCP architecture for national statistics

The AI host communicates through an MCP client using structured JSON-RPC messages to a national statistics MCP server.

Tools

Executable actions such as listing datasets, comparing geographies, validating claims, and creating a plain-language summary.

Resources

Approved read-only context such as dataset metadata, subject definitions, quality rules, and disclosure warnings.

Prompts

Reusable instruction templates for briefing notes, data-quality checks, source notes, and safe public communication.

      Privacy, security and statistical quality gates

      Before the tool call

      • Authenticate the user and propagate least-privilege authorization.
      • Expose only approved public or permissioned aggregate datasets.
      • Require dataset ID, geography, reference period and unit before comparison.
      • Block personal, confidential, protected or unpublished data in a public assistant.

      After the tool call

      • Return source metadata, limitations and quality notes with every answer.
      • Do not infer causes from descriptive statistics without supporting data.
      • Log tool calls for review, but avoid logging sensitive user content.
      • Mark AI summaries as draft text that requires human verification.

      Responsible-use boundary

      This prototype allows an AI assistant to support analysis. It does not automate eligibility, enforcement, benefit, compliance or other decisions about people. Production use would require departmental privacy, security, accessibility, official languages and statistical-methodology review.
