MCP 2.0-ready authoritative national statistics

This prototype shows how a national statistical organization can expose authoritative statistical information to AI assistants using MCP semantics: lifecycle negotiation, resources, prompts, tools, structured outputs, client features, consent, logging, error handling and quality controls.

Static HTML/CSS/JavaScript prototype · No login · No backend · No external API calls · Demo data only

Important protocol note

The current public MCP specification is versioned by date, not by a formal “MCP 2.0” release label. This page uses “MCP 2.0-ready” as a presentation profile for MCP 2025-06-18 plus JSON-RPC 2.0 semantics, and explicitly marks where production implementation would need specification validation.

Authoritative-source concept

An AI assistant should not infer that a table, chart or third-party page is authoritative. The NSO MCP server validates publisher, release state, dataset identifier, geography, reference period, unit, methodology note and citation text before analysis.

MCP architecture for authoritative national statistics

      UN/NSO alignment for authoritative information sharing

      This MCP pattern supports the official-statistics role of national statistical organizations by making authority, methodology, quality and metadata explicit before AI-assisted use.

      International sharing pattern

      NSO release authority
      →
      Metadata and quality statement
      →
      SDMX/API dataflow
      →
      UN, partners and trusted AI access

      Quality, privacy and security gates

      Before a tool call

      • Confirm user consent and intended use.
      • Validate source authority, official release state and permitted access.
      • Require dataset ID, geography, reference period, unit, method and citation.
      • Block personal, protected, confidential, embargoed or unpublished data in public contexts.

      After a tool call

      • Return structured content plus human-readable text.
      • Record audit event, latency and source coverage metric.
      • Flag unsupported causality, ranking, deficit framing and missing limitations.
      • Require human verification before public release or decision support.

      Responsible-use boundary

      This prototype supports statistical discovery and explanation only. It does not automate eligibility, enforcement, benefit, compliance or decisions about people.
